# S3 – Inbetriebnahme

## Ziel
Beschreibung, wie WG-ShopSync initial bereitgestellt und in Betrieb genommen wird.

## Deployment-Schritte
1. Backend (Node.js) wird auf [Plattform] deployed
2. Cloud-Datenbank wird initialisiert
3. Mobile App wird über App Store / TestFlight / APK verteilt
4. Umgebungsvariablen (.env) werden gesetzt (nie im Repo!)

## Voraussetzungen
- Flutter SDK installiert
- Node.js + npm installiert
- Zugang zur Cloud-DB

## Rollout-Strategie
- Zunächst interner Test mit dem Projektteam
- Danach Freigabe für Testnutzer (Beta)
